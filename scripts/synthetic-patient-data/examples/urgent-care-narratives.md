Synthetic Urgent Care Patient Narratives
=========================================

Fifteen narratives covering a range of ages and typical urgent-care presentations.
Each is detailed enough to fully populate the EHR's visit-detail screen and
progress note: demographics, contact, insurance, responsible party, emergency
contact, PCP, allergies, medications, past medical/surgical history,
chief complaint, HPI, vitals, exam findings, diagnoses, orders, and disposition.

For minors, the responsible party is the parent and consent-signer relationship
reflects that. For adults, responsible party is "self".

Insurance carriers used are limited to those typically copied into a synth
project's payer-Organization catalog: Aetna, Blue Cross Blue Shield, Cigna,
UnitedHealthcare, Humana, Anthem, Kaiser, Medicare, Medicaid, Tricare.

All visits are in-person urgent-care walk-ins. Visit dates are unspecified —
the synthesis pipeline auto-bumps any past date to the next available
15-minute slot.

Notes for the synthesis pipeline
-------------------------------------------------------------------------------

When converting these narratives to scenario JSONs, remember:

- `reasonForVisit` must use a value from the urgent-care option list
  (BOOKING_CONFIG.serviceCategories) before the " - " separator. The
  free-text additional detail follows after. e.g.
  `"Throat pain - sore throat × 3 days, fever, exudate"`. An
  unrecognized leading value renders as the literal "undefined" on the
  visit-detail page.

- For minors, the consent-signer relationship and responsible party
  reflect parent. The harvest's `account-coverage` and `responsible-party`
  pages handle this from the QR.

- Each insurance carrier name should match an existing payer
  Organization in the synth project (copied via copy-payer-organizations.ts).
  If an exact match is missing, the closest carrier name will be
  resolved.

- The pipeline is idempotent on Patient (deterministic synth identifier
  derived from name + DOB + sex). Each rerun for the same patient
  reuses the Patient and creates a fresh Appointment + Encounter.

- Charge-master and fee-schedule entries seeded via the RCM zambdas
  are shared across all visits. CPT codes referenced in the narratives
  will resolve to the prices the synth pipeline sets up
  (99213 office visit, 87880 strep test, 71046 chest X-ray, 90471
  immunization administration). New CPT codes can be added to the
  scenario's `pricing.cptPrices` list.
